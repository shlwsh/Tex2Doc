\paragraph{Vision 接口契约。}
网关以子进程调用 \texttt{edge\_iqa.cli}，\texttt{PYTHONPATH} 指向 \texttt{doctor/paper1}。
响应包含 \texttt{q\_img}、\texttt{flags}、\texttt{t\_iqa\_ms}，以及 \texttt{recommended\_tau.json} 中的 \texttt{threshold\_tau}。
子进程隔离可避免 NumPy/PIL 阻塞 \texttt{rclpy} 执行器。

\subsection{离线实验驱动}
ShezhenV3 验证的推荐顺序为：
(1) \texttt{import\_shezhenv3.py}（写入 \texttt{bboxes}）；
(2) 可选 \texttt{train\_b3\_mobilenet.py}；
(3) \texttt{calibrate\_tau\_tcm.py}（ROI）；
(4) \texttt{roi\_ablation.py}；
(5) \texttt{run\_matrix\_tcm.py}（B2 Edge-IQA + B3 学习分数）；
(6) 图表与 \LaTeX{} 生成。
一键脚本 \texttt{paper1\_run\_tcm\_full.sh} 链式调用上述步骤并编译 PDF。

\subsection{ROI 与 B3 模块}
\texttt{coco\_roi.py} 实现并集框 + padding 裁剪；\texttt{scorer.py} 与 \texttt{learned\_b3.py} 共用 ROI 逻辑。
B3 权重为 \texttt{b3\_mobilenet.pt}，验证准确率 98.34\%（1144 对）。

\paragraph{矩阵打分。}
每测试帧预计算 \texttt{q\_b2} 与 \texttt{q\_b3}；B0--B2 使用 Edge-IQA 分数，B3 使用学习概率，避免重采 uplift 被重复计入。

\paragraph{目录与 CI。}
主要目录：\texttt{edge\_iqa/}、\texttt{langgraph\_router/}、\texttt{experiments/splits|results|edge\_iqa/}、\texttt{latex/}、\texttt{franka\_api\_server/}。
CI 运行单元测试；全量 ShezhenV3 验证为本地文稿目标，步骤见 \texttt{REPRODUCE.md}。
修改 ROI padding 或 Edge-IQA 权重后，须重跑 orchestrator 再更新表 II。

\paragraph{数据集路径。}
Windows 路径 \texttt{D:\textbackslash ...\textbackslash shezhenv3-coco} 在 WSL 下映射为 \texttt{/mnt/d/...}，仅需修改 \texttt{tcm\_paths.yaml} 中的 \texttt{dataset\_root}。
\texttt{test\_scorer.py}、\texttt{test\_routing.py} 可在无 TCM 数据挂载的 CI 环境运行。
