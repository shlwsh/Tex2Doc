## 一、项目概述

### 1.1 背景

Tex2Doc 目前存在三套客户端，各自独立管理版本：

| 客户端 | 当前版本检查 | 现状 |
|--------|-------------|------|
| Slint 桌面端（`crates/desktop-slint`） | 有（`GET /v1/releases/{channel}`，返回 mock 数据） | 版本检查已实现，下载/安装未实现 |
| Flutter Web 端（`flutter_app/`） | 无 | 完全未实现 |
| Flutter 桌面端（`flutter_app/`） | 无 | 完全未实现 |

服务端当前 `/v1/releases/:channel` 接口返回硬编码的 mock 数据，未接入真实存储。数据库 Schema 中已有 `release_manifests` 表定义（见 `docs-zh/money/001_docdb_business_schema.sql`），但未连接任何服务端代码。

### 1.2 目标

构建一套完整的跨平台升级管理系统：

- **服务端**：升级包存储元数据管理（上传、查询、策略配置）、灰度发布、强制升级、回滚、发布历史审计
- **Slint 桌面端**：在已有版本检查（P9）基础上，完善下载、安装、后台检查
- **Flutter 端**：从零实现版本检查、下载、安装（Web 通过浏览器更新，桌面端类似 Slint）
- **CI/CD**：自动化构建 → 签名 → 上传 → 触发发布 → 灰度 rollout

### 1.3 范围

| 分层 | 内容 |
|------|------|
| 数据库 | PostgreSQL `release_manifests`、`release_strategies`、`release_rollout_events`、`release_audit_log` 表 |
| 服务端 | Axum HTTP API（CRUD 发布管理 + 客户端查询） |
| Slint 桌面端 | 完善自动升级（`docs-zh/slint/Tex2Doc-Slint桌面端自动升级功能设计方案-20260623.md`） |
| Flutter 端 | 新增版本检查 API 集成 |
| CI/CD | GitHub Actions 发布流水线 |

## 二、数据库设计

### 2.1 表结构总览

```
release_manifests          — 升级包元数据
release_strategies          — 发布策略（强制/可选/灰度）
release_rollout_events      — 灰度 rollout 事件记录
release_audit_log           — 发布操作审计日志
app_update_preferences      — 用户端升级偏好（可选）
```

### 2.2 核心表设计

#### `release_manifests`（升级包元数据）

已有定义（`docs-zh/money/001_docdb_business_schema.sql`），本方案扩展字段：

```sql
CREATE TABLE IF NOT EXISTS release_manifests (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    -- 版本标识
    channel TEXT NOT NULL CHECK (channel IN ('stable', 'beta', 'dev')),
    platform TEXT NOT NULL CHECK (platform IN ('windows', 'macos', 'linux', 'web', 'android', 'ios')),
    arch TEXT NOT NULL DEFAULT 'x64',
    version TEXT NOT NULL,  -- 语义化版本: "1.27.0"

    -- 文件信息
    download_url TEXT NOT NULL,
    sha256 TEXT NOT NULL,
    file_size_bytes BIGINT NOT NULL DEFAULT 0,
    signature TEXT NOT NULL DEFAULT '',
    signature_algorithm TEXT NOT NULL DEFAULT 'sha256',  -- sha256 | gpg | ossl

    -- 发布信息
    release_notes TEXT NOT NULL DEFAULT '',
    release_title TEXT NOT NULL DEFAULT '',
    published_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    published_by TEXT NOT NULL DEFAULT 'system',
    is_prerelease BOOLEAN NOT NULL DEFAULT false,

    -- 生命周期
    active BOOLEAN NOT NULL DEFAULT true,
    deprecated_at TIMESTAMPTZ,
    deprecated_by TEXT,
    deprecation_reason TEXT,

    -- 唯一约束（同一渠道+平台+架构+版本唯一）
    UNIQUE (channel, platform, arch, version)
);

CREATE INDEX idx_releases_channel_platform ON release_manifests(channel, platform, active);
CREATE INDEX idx_releases_version ON release_manifests(version DESC);
```

#### `release_strategies`（发布策略）

```sql
CREATE TABLE IF NOT EXISTS release_strategies (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    -- 关联 release_manifests
    release_id UUID NOT NULL REFERENCES release_manifests(id) ON DELETE CASCADE,

    -- 策略类型
    strategy_type TEXT NOT NULL CHECK (strategy_type IN (
        'optional',       -- 可选升级（默认）
        'recommended',    -- 推荐升级（客户端提示但不强制）
        'force',          -- 强制升级（客户端必须升级才能继续使用）
        'grayroll'        -- 灰度发布（按百分比/用户群 rollout）
    )),

    -- 强制升级配置
    min_required_version TEXT,  -- 低于此版本的客户端必须升级（如 "1.25.0"）
    force_deadline_at TIMESTAMPTZ,  -- 强制升级截止时间（超过后阻断使用）
    block_if_outdated BOOLEAN NOT NULL DEFAULT false,

    -- 灰度发布配置
    rollout_percentage INTEGER NOT NULL DEFAULT 100 CHECK (rollout_percentage BETWEEN 0 AND 100),
    rollout_target_users TEXT[],  -- 指定用户 ID 白名单（为空表示不限）
    rollout_target_regions TEXT[],  -- 指定地区白名单（为空表示不限）
    rollout_start_at TIMESTAMPTZ,
    rollout_end_at TIMESTAMPTZ,

    -- 升级提示文案
    prompt_title TEXT NOT NULL DEFAULT '',
    prompt_message TEXT NOT NULL DEFAULT '',
    prompt_dismissable BOOLEAN NOT NULL DEFAULT true,  -- 强制升级时为 false

    -- 元数据
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    created_by TEXT NOT NULL DEFAULT 'system',
    updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_by TEXT,
    is_active BOOLEAN NOT NULL DEFAULT true,

    UNIQUE (release_id, strategy_type)
);
```

#### `release_rollout_events`（灰度 rollout 事件记录）

```sql
CREATE TABLE IF NOT EXISTS release_rollout_events (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    release_id UUID NOT NULL REFERENCES release_manifests(id) ON DELETE CASCADE,
    strategy_id UUID NOT NULL REFERENCES release_strategies(id) ON DELETE CASCADE,

    -- 事件类型
    event_type TEXT NOT NULL CHECK (event_type IN (
        'rollout_started',
        'rollout_percentage_updated',
        'rollout_paused',
        'rollout_resumed',
        'rollout_completed',
        'rollout_aborted',
        'rollback_triggered'
    )),

    -- 事件详情
    previous_percentage INTEGER,
    new_percentage INTEGER,
    event_reason TEXT,
    triggered_by TEXT NOT NULL DEFAULT 'system',

    created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_rollout_events_release ON release_rollout_events(release_id, created_at DESC);
```

#### `release_audit_log`（审计日志）

```sql
CREATE TABLE IF NOT EXISTS release_audit_log (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    actor TEXT NOT NULL,  -- 操作人（GitHub actor / admin user）
    action TEXT NOT NULL,  -- publish | deprecate | update_strategy | rollback | grayroll_adjust
    target_release_id UUID REFERENCES release_manifests(id),

    -- 操作详情
    details JSONB NOT NULL DEFAULT '{}',
    -- 示例: {"previous_version": "1.26.0", "new_version": "1.27.0", "channel": "stable"}

    ip_address INET,
    user_agent TEXT,

    created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_audit_actor ON release_audit_log(actor, created_at DESC);
CREATE INDEX idx_audit_target ON release_audit_log(target_release_id, created_at DESC);
```

#### `app_update_preferences`（用户端升级偏好，可选）

```sql
CREATE TABLE IF NOT EXISTS app_update_preferences (
    user_id TEXT NOT NULL,  -- 可为空（匿名用户）
    device_id TEXT NOT NULL,  -- 设备唯一标识

    -- 偏好设置
    auto_update BOOLEAN NOT NULL DEFAULT true,
    update_channel TEXT NOT NULL DEFAULT 'stable'
        CHECK (update_channel IN ('stable', 'beta', 'dev')),
    metered_update_allowed BOOLEAN NOT NULL DEFAULT false,
    auto_restart BOOLEAN NOT NULL DEFAULT false,

    -- 上报信息（用于灰度决策）
    platform TEXT NOT NULL,
    app_version TEXT NOT NULL,
    last_seen_at TIMESTAMPTZ NOT NULL DEFAULT now(),

    PRIMARY KEY (user_id, device_id)
);
```

**灰度决策逻辑**（服务端）：

```rust
async fn decide_rollout(
    release_id: Uuid,
    user_id: Option<&str>,
    device_id: Option<&str>,
    platform: &str,
    app_version: &str,
) -> Option<ReleaseManifest> {
    let strategy = find_active_strategy(release_id).await?;

    match strategy.strategy_type.as_str() {
        "optional" | "recommended" => {
            Some(get_manifest_by_id(release_id).await)
        }
        "force" => {
            Some(get_manifest_by_id(release_id).await)
        }
        "grayroll" => {
            // 1. 白名单用户直接通过
            if strategy.rollout_target_users.contains(&user_id.to_string()) {
                return Some(get_manifest_by_id(release_id).await);
            }
            // 2. 白名单地区检查（略）
            // 3. 百分比灰度
            let user_hash = hash_for_rollout(user_id, device_id);
            let bucket = user_hash % 100;
            if bucket < strategy.rollout_percentage {
                Some(get_manifest_by_id(release_id).await)
            } else {
                None  // 灰度未命中，返回当前版本（不提示更新）
            }
        }
        _ => None,
    }
}
```

## 四、灰度发布决策模型

### 4.1 灰度百分比计算

为保证同一用户始终命中同一灰度桶，使用确定性哈希：

```rust
fn hash_for_rollout(user_id: Option<&str>, device_id: Option<&str>) -> u32 {
    let input = format!("{}_{}", user_id.unwrap_or("anonymous"), device_id.unwrap_or(""));
    let bytes = sha2::Sha256::digest(input.as_bytes());
    // 取前 4 字节作为 u32
    u32::from_le_bytes(bytes[0..4].try_into().unwrap())
}

fn in_rollout_bucket(user_id: Option<&str>, device_id: Option<&str>, percentage: u32) -> bool {
    let hash = hash_for_rollout(user_id, device_id);
    let bucket = (hash % 100) as u32;
    bucket < percentage
}
```

> 优点：用户 ID 不变时，灰度百分比调高后，原来命中的用户仍然命中；调低后，原来命中的部分用户会退出。不依赖服务端状态存储。

### 4.2 灰度阶段生命周期

```
  0% ─── 10% ─── 25% ─── 50% ─── 100%
   │       │       │       │       │
  初始     扩大    扩大    全量    完成
  小范围   友好    核心    推送    稳定
```

| 阶段 | 百分比 | 观察重点 |
|------|--------|---------|
| 内部测试 | 0%（白名单） | 功能验收 |
| 灰度 10% | 10% | 核心错误率、崩溃率 |
| 扩大 25% | 25% | 性能指标、网络抖动 |
| 核心 50% | 50% | 用户满意度、NPS |
| 全量 100% | 100% | 稳定性确认 |

### 4.3 强制升级截止时间

当 `strategy_type = force` 且设置了 `force_deadline_at`：

- 客户端检查到当前时间 >= `deadline_at` 时，弹出不可关闭的阻断弹窗
- Slint 端：禁用所有功能按钮，显示 `AlertBanner` 满屏遮罩
- Flutter Web 端：覆盖全屏，显示倒计时

### 5.3 灰度决策实现

```rust
// release_rollout.rs

pub async fn decide_rollout(
    repo: &ReleaseRepository,
    channel: &str,
    platform: &str,
    arch: &str,
    user_id: Option<&str>,
    device_id: Option<&str>,
    app_version: &str,
) -> Result<Option<ClientReleaseResponse>, ReleaseError> {
    // 1. 获取最新 active manifest
    let manifest = repo
        .find_latest_active(channel, platform, arch)
        .await?
        .ok_or(ReleaseError::NoActiveRelease)?;

    // 2. 获取 active 策略
    let strategy = repo.find_active_strategy(manifest.id).await?;

    // 3. 根据策略类型决策
    match strategy.as_ref().map(|s| s.strategy_type) {
        Some(StrategyType::Optional) | Some(StrategyType::Recommended) | Some(StrategyType::Force) => {
            Ok(Some(build_client_response(&manifest, strategy.as_ref())))
        }
        Some(StrategyType::Grayroll) => {
            let s = strategy.as_ref().unwrap();

            // 时间窗口检查
            if let Some(start) = s.rollout_start_at {
                if chrono::Utc::now() < start { return Ok(None); }
            }
            if let Some(end) = s.rollout_end_at {
                if chrono::Utc::now() > end { return Ok(None); }
            }

            // 白名单用户检查
            if !s.rollout_target_users.is_empty() {
                if let Some(uid) = user_id {
                    if !s.rollout_target_users.contains(&uid.to_string()) {
                        return Ok(None);
                    }
                } else {
                    return Ok(None);  // 需要登录但未登录，不灰度
                }
            }

            // 百分比灰度
            let in_bucket = in_rollout_bucket(user_id, device_id, s.rollout_percentage as u32);
            if in_bucket {
                Ok(Some(build_client_response(&manifest, Some(s))))
            } else {
                Ok(None)  // 未命中灰度桶，不返回更新
            }
        }
        None => Ok(Some(build_client_response(&manifest, None))),
    }
}

fn in_rollout_bucket(user_id: Option<&str>, device_id: Option<&str>, percentage: u32) -> bool {
    let input = format!(
        "{}_{}",
        user_id.unwrap_or("anonymous"),
        device_id.unwrap_or("unknown")
    );
    let digest = sha2::Sha256::digest(input.as_bytes());
    let hash = u32::from_le_bytes(digest[0..4].try_into().unwrap());
    (hash % 100) as u32 < percentage
}
```

### 5.4 回滚实现

```rust
// release_service.rs

pub async fn rollback_release(
    repo: &ReleaseRepository,
    audit: &AuditLog,
    release_id: Uuid,
    actor: &str,
    reason: &str,
) -> Result<RollbackResult, ReleaseError> {
    // 1. 获取当前版本信息
    let current = repo.find_by_id(release_id).await?
        .ok_or(ReleaseError::NotFound)?;

    // 2. 查找上一个版本（同渠道+平台+架构，active 且 version < current.version）
    let previous = repo.find_previous_active(
        &current.channel,
        &current.platform,
        &current.arch,
        &current.version,
    ).await?.ok_or(ReleaseError::NoPreviousRelease)?;

    // 3. 事务：停用当前 + 激活上一个
    repo.transaction(|tx| async {
        tx.deactivate_release(release_id).await?;
        tx.activate_release(previous.id).await?;
        Ok(())
    }).await?;

    // 4. 迁移策略（复制当前策略到上一个版本）
    let strategy = repo.find_active_strategy(release_id).await?;
    if let Some(s) = strategy {
        let new_strategy = ReleaseStrategyDb {
            id: Uuid::new_v4(),
            release_id: previous.id,
            ..s
        };
        repo.upsert_strategy(&new_strategy).await?;
    }

    // 5. 记录审计
    audit.log(AuditEntry {
        actor: actor.to_string(),
        action: "rollback".to_string(),
        target_release_id: Some(release_id),
        details: json!({
            "rolled_back_from": current.version,
            "rolled_back_to": previous.version,
            "reason": reason,
        }),
    }).await?;

    Ok(RollbackResult {
        rolled_back_from: current.version,
        rolled_back_to: previous.version,
        previous_strategy: strategy,
    })
}
```

```dart
// flutter_app/lib/update_service.dart

class UpdateService {
  final CommercialApiClient _api;
  final String _currentVersion;
  final String _platform;

  UpdateService(this._api, this._currentVersion, this._platform);

  /// 后台静默检查（应用启动时调用）
  Future<UpdateState> checkSilently({String? userId, String? deviceId}) async {
    try {
      final release = await _api.checkRelease(
        channel: 'stable',
        appVersion: _currentVersion,
        platform: _platform,
        userId: userId,
        deviceId: deviceId,
      );
      if (release == null) return UpdateState.upToDate();
      return _evaluate(release);
    } catch (e) {
      return UpdateState.error(e.toString());
    }
  }

  UpdateState _evaluate(ReleaseInfo release) {
    if (release.strategy == null) {
      return UpdateState.available(release);
    }
    final s = release.strategy!;
    if (s.isForce) {
      // 检查截止时间
      if (s.forceDeadlineAt != null &&
          DateTime.now().isAfter(s.forceDeadlineAt!)) {
        return UpdateState.blocked(release, s.promptMessage);
      }
    }
    return UpdateState.available(release);
  }
}

sealed class UpdateState {
  factory UpdateState.upToDate() = UpdateStateUpToDate;
  factory UpdateState.available(ReleaseInfo release) = UpdateStateAvailable;
  factory UpdateState.blocked(ReleaseInfo release, String message) = UpdateStateBlocked;
  factory UpdateState.error(String message) = UpdateStateError;
}
```

```yaml
        shell: bash

  # Job 2: 上传到 CDN/对象存储 + 发布到 API
  # ============================================================
  publish:
    needs: build
```

## 十、风险与缓解

| 风险 | 等级 | 缓解措施 |
|------|------|---------|
| 灰度决策算法不一致（服务端 vs 客户端） | **高** | 使用确定性哈希（SHA256），两端算法完全相同 |
| 回滚后旧版本策略配置丢失 | **中** | 回滚时自动将原策略迁移到上一版本 |
| CI/CD 发布时 OSS 上传成功但 API 发布失败 | **中** | API 发布失败时 CI 标记失败，OSS 文件需手动清理 |
| Admin Token 泄露 | **高** | 使用 GitHub Secrets 管理，不硬编码；生产环境迁移到 Vault/OIDC |
| 客户端降级攻击（下载旧版被篡改的安装包） | **高** | SHA256 校验（强依赖）；未来接入签名验证 |
| 强制升级截止时间漂移（客户端与服务端时区差异） | **低** | 截止时间统一使用 UTC，服务端返回 ISO8601 UTC 字符串 |
| PostgreSQL 单点故障（升级 API 不可用） | **中** | 客户端降级：超时不阻塞（timeout 10s），可继续使用旧版 |

## 十一、验收标准

| # | 标准 |
|---|------|
| 1 | 服务端 `GET /v1/releases/stable` 返回真实数据库中的最新 stable 版本 |
| 2 | `POST /admin/releases` 成功发布新版本，旧版本自动 `active=false` |
| 3 | `POST /admin/releases/{id}/rollback` 将新版本 `active=false`，上一版本 `active=true` |
| 4 | 灰度 10% 时，同一 user_id 的用户始终命中同一灰度桶（多次请求结果一致） |
| 5 | Flutter Web 启动时后台检查更新，有更新时显示顶部 banner |
| 6 | Slint 桌面端收到 `strategy.type=force` 时，强制升级弹窗不可关闭 |
| 7 | CI/CD `workflow_dispatch` 触发后自动构建、上传、调用 API 发布 |
| 8 | Admin API 所有操作记录到 `release_audit_log` |
| 9 | 审计日志可通过 `GET /admin/audit` 按 actor/action/时间范围查询 |
| 10 | `cargo build -p doc-server` 编译通过 |
| 11 | Flutter `flutter build web` 编译通过 |
