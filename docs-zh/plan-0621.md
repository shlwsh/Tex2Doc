## P1: ActiveProfile 全链路重构（1-2 周）

```rust
pub enum ProfileRef {
    Auto,
```

### P1.3 改造 Profile 解析链路

**文件**：`crates/compiler-engine/src/profiles.rs`
- `resolve_active_profile(options, vfs) → ActiveProfile`：按 `ProfileRef` 解析，若为 `Auto` 则调用 `JournalDetector`，若为 `Id` 则加载对应 TOML，保留 `Legacy` 兼容路径

### P1.5 修复 Profile 映射降级

- `semantic-convert --profile tacl` → 加载 `profiles/tacl.toml`（而非降级到 `JosPaper`）

- `semantic-convert --profile nature` → 加载 `profiles/nature.toml`

### P1.6 报告中包含 ActiveProfile

**文件**：`crates/compiler-engine/src/lib.rs` — `CompileReport`

## P2: Profile-aware RuleEngine 接入（1 周）

### P2.1 实现 `build_rule_engine(active_profile)`

**文件**：`crates/compiler-engine/src/lib.rs`（或新建 `rule_engine_integration.rs`）

```rust
fn build_rule_engine(active_profile: &ActiveProfile) -> RuleEngine {
    let mut engine = RuleEngine::new();
    // 加载内置期刊规则
    for rule in journal_rules(&active_profile.id) {
        engine.registry_mut().register(rule);
    }
    // 加载 Profile TOML 中的 macro_rules
    for rule in toml_macro_rules(&active_profile.spec.macro_rules) {
        engine.registry_mut().register(rule);
    }
    engine
}
```

## P3: QualityGate V2（1-2 周）

### P3.1 新增 `QualityGateResult` V2 类型

**文件**：`crates/compiler-engine/src/lib.rs`

```rust
pub struct QualityGateResult {
    pub status: QualityStatus,
    pub score: u8,
    pub checks: Vec<QualityCheck>,
    pub blocking_issues: Vec<QualityIssue>,
    pub warnings: Vec<QualityIssue>,
}

pub enum QualityStatus {
    Passed,
    PassedWithWarnings,
    Failed,
}
```

### P3.2 新增检查项

| 检查 | 阈值来源 | 等级 |
|---|---|---|
| compatibility score | `CompatibilityReport.score` | Error |
| unresolved references | `ReferenceGraph` | Error/Warning |
| raw fallback count | `Document` | Error |
| missing images | `ImageAssets` | Error |
| docx zip structure | DOCX bytes | Error |
| style coverage | style_map + document.xml | Warning |
| profile confidence | `JournalDetector` | Warning |
| runtime fallback | `BackendReport` | Warning |
| DOCX openable | zip 格式校验 | Error |

### P3.3 Profile TOML quality 配置

**文件**：`crates/compiler-engine/profiles/*.toml`

## 风险与缓解

| 风险 | 缓解策略 |
|---|---|
| P1 ActiveProfile 重构影响现有编译链路 | 先写集成测试，再改核心类型 |
| P7 TeX runtime sandbox 实现复杂度 | 先用 Docker 容器方案，快速验证 |
| P9 三平台签名分发 | 使用 sigstore/cosign 简化签名流程 |
| 真实样本收集耗时 | 优先收集公开域论文（如 arXiv），确保隐私合规 |
