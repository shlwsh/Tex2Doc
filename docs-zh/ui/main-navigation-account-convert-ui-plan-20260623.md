# Tex2Doc 主页面账号与转换模块 UI 设计方案
> **版本 / Version**: v2.0
> **最后更新日期 / Last Updated**: 2026-06-26



日期：2026-06-23

## 背景

当前 Web 主页面左侧已有“工作台 / 账号 / 转换”导航，但导航项只是静态展示，点击“账号”和“转换”不会切换内容区，用户会误以为模块不可用。商业化推广阶段需要让首屏导航、账号登录、云端语义转换形成完整闭环。

## 设计目标

1. 左侧导航可点击，清晰反馈当前所在模块。
2. “账号”模块可独立进入，完成注册、登录、刷新额度、查看套餐。
3. “转换”模块可独立进入，复用账号模块登录状态，上传 ZIP 后调用云端语义引擎生成 DOCX。
4. “工作台”保留总览形态，同时展示账号和转换双卡片，适合演示和日常使用。
5. 小屏或窄窗口下没有左侧栏时，内容区顶部提供分段切换控件。

## 信息架构

主页面拆分为三个一级模块：

| 模块 | 入口 | 内容 |
| --- | --- | --- |
| 工作台 | 左侧“工作台” | 指标概览 + 账号卡片 + 转换卡片 |
| 账号 | 左侧“账号” | 云端账号卡片，管理 API 地址、注册、登录、额度、套餐 |
| 转换 | 左侧“转换” | 文档转换卡片，选择 ZIP、填写主 TeX、调用云端语义引擎、下载 DOCX |

## 交互方案

- 左侧导航项使用按钮语义，支持鼠标点击和辅助功能识别。
- 当前导航项显示主色浅底和主色图标。
- 切换模块不刷新页面，不丢失已登录 access token。
- 登录或注册成功后，账号模块把 API 地址和 access token 提升到页面 shell 状态。
- 转换模块读取 shell 中的 API 地址和 access token。
- 未登录时点击“开始转换”，提示先注册或登录，不再静默使用低质量本地 WASM 路径。
- 登录后点击“开始转换”，固定调用云端语义引擎链路：
  - `POST /v1/uploads`
  - `POST /v1/conversions`
  - `GET /v1/conversions/{id}`
  - `GET /v1/conversions/{id}/download/docx`

## 响应式方案

- 桌面宽度：左侧固定导航 + 顶部工具条 + 内容区。
- 窄屏宽度：隐藏左侧导航，在内容区顶部显示分段按钮，提供“工作台 / 账号 / 转换”切换。
- 卡片不再嵌套卡片，模块内容保持单层卡片结构。

## 验收标准

1. 点击左侧“账号”后，内容区只显示云端账号模块。
2. 点击左侧“转换”后，内容区只显示文档转换模块。
3. 点击左侧“工作台”后，内容区显示指标概览和账号/转换双卡片。
4. 注册或登录后切换到转换模块，转换模块可使用云端 token。
5. 未登录时转换给出明确提示。
6. 上传 `examples/paper3/upload.zip` 后，转换请求必须进入云端语义引擎 API。
7. Playwright 集成测试需要覆盖导航可点击性和云端转换链路。

## 当前实现结果

- `_WorkspaceShell` 改为有状态 shell，持有当前模块、API 地址和 access token。
- `_Sidebar` 和 `_NavItem` 改为可点击导航。
- `_WorkspaceContent` 根据当前模块渲染不同内容。
- `_SectionTabs` 为窄屏提供分段切换。
- `_ResponsiveCards` 改为接收 shell 状态，账号和转换双卡片共享登录态。
