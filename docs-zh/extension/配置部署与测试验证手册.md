## 环境准备

### 系统要求

| 项目 | 要求 |
|------|------|
| **操作系统** | Windows 10+, macOS 10.15+, Linux (Ubuntu 20.04+) |
| **Node.js** | >= 18.0.0 |
| **npm** | >= 9.0.0 |
| **浏览器** | Chrome 88+, Edge 88+, Firefox 78+, Safari 15+ |

### 必需工具

```bash
# Node.js (推荐使用 nvm 管理版本)
# 安装 nvm (Linux/macOS)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 18
nvm use 18

# Windows 使用 nvm-windows
# https://github.com/coreybutler/nvm-windows/releases
```

```bash

# https://rustup.rs/
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# wasm-pack (仅在重新编译 WASM 时需要)
cargo install wasm-pack
```

## 项目配置

### 1. 安装依赖

```bash
# 进入扩展目录
cd apps/browser-extension

# 安装依赖
npm install
```

### 2. 环境变量配置

创建 `.env.local` 文件配置本地开发环境：

```bash
# API 配置
API_BASE_URL=https://api.tex2doc.cn
API_VERSION=v1

# 开发模式配置
DEV_MODE=true
LOG_LEVEL=debug

# 可选：本地 API 服务器（开发时使用）
# API_BASE_URL=http://localhost:8080
```

### 3. WXT 配置文件

`wxt.config.ts` 是扩展的核心配置文件：

```typescript
import { defineConfig } from 'wxt';

export default defineConfig({
  // 源码目录
  srcDir: './src',

  // 输出目录基础路径
  outBaseDir: '.output',

  // 输出目录模板
  outDirTemplate: "{{browser}}-mv{{manifestVersion}}{{modeSuffix}}",

  // 入口点目录
  entrypointsDir: './src/entrypoints',

  // 公共资源目录
  publicDir: './public',

  // 路径别名
  alias: {
    '@': './src',
    '@api': './src/api',
    '@browser': './src/browser',
    '@conversion': './src/conversion',
    '@state': './src/state',
    '@ui': './src/ui',
    '@shared': './src/shared',
  },

  // manifest 配置
  manifest: ({ browser, mode }) => ({
    name: 'Tex2Doc - LaTeX to Word',
    version: '0.1.0',
    description: 'Convert LaTeX documents to Word (.docx) directly in your browser',
    permissions: [
      'storage',
      'downloads',
      'contextMenus',
      'notifications',
      'alarms',
    ],
    host_permissions: ['https://api.tex2doc.cn/*'],
    optional_host_permissions: [
      'https://www.overleaf.com/*',
      'https://*.overleaf.com/*',
      'https://arxiv.org/*',
      'https://*.arxiv.org/*',
    ],
    // 针对不同浏览器的特殊配置
    ...(browser === 'safari' && {
      permissions: [
        'storage',
        'downloads',
        'contextMenus',
        'notifications',
      ],
    }),
  }),

  // 开发服务器配置
  devServer: {
    port: 3000,
    headless: false,
  },

  // ZIP 打包配置
  zip: {
    outputPath: '{{extensionOutputDir}}/tex2doc-{{browser}}.zip',
  },
});
```

## 本地开发

### 启动开发服务器

```bash
# 方式一：在 browser-extension 目录下直接运行
cd apps/browser-extension
npm run dev

# 方式二：从项目根目录运行
npm run extension:dev
```

### 针对特定浏览器开发

```bash
# Chrome 开发
npm run dev:chrome

# Firefox 开发
npm run dev:firefox

# Safari 开发
npm run dev:safari
```

### 开发工作流

1. **启动开发服务器**
   ```bash
   npm run dev
   ```
   输出示例：
   ```
   🚀 Extension Dev Server running at:
   - Chrome: chrome://extensions/?Developer mode → Load unpacked → .output/chrome-mv3-dev/
   ```

2. **加载扩展到浏览器**
   - Chrome/Edge: 访问 `chrome://extensions/`，开启开发者模式，加载 `.output/chrome-mv3-dev/`
   - Firefox: 访问 `about:debugging#/runtime/this-firefox`，临时加载

3. **修改代码**
   - WXT 支持热重载，保存后自动重新构建
   - 部分修改可能需要手动刷新扩展（点击扩展页面的刷新按钮）

4. **调试**
   - 右键点击扩展图标 → 检查
   - 或右键点击网页 → 检查 → Console

### WASM 开发（可选）

仅在需要修改 WASM 引擎时执行：

```bash
# 构建 WASM 引擎
npm run build:wasm:extension

# 或仅构建 debug 版本（更快）
cd crates/wasm
wasm-pack build --target web --out-dir ../../apps/browser-extension/public/wasm --out-name doc_engine --dev
```

## 构建部署

### 构建流程

```bash
# 1. 构建 WASM 引擎（仅首次或更新 WASM 时需要）
npm run build:wasm:extension

# 2. 构建扩展
npm run extension:build

# 3. 修复 Chrome manifest（如需要）
node scripts/fix-chrome-manifest.mjs
```

### 针对不同浏览器构建

| 浏览器 | 构建命令 | 输出目录 | 清单版本 |
|--------|----------|----------|----------|
| Chrome | `npm run build:chrome` | `.output/chrome-mv3/` | MV3 |
| Edge | `npm run build:edge` | `.output/chrome-mv3-edge/` | MV3 |
| Firefox | `npm run build:firefox` | `.output/firefox-mv2/` | MV2 |
| Safari | `npm run build:safari` | `.output/safari-mv2/` | MV2 |

### 构建后验证

检查构建输出：

```bash
# 列出所有构建产物
ls -la .output/

# 验证 manifest.json
cat .output/chrome-mv3/manifest.json | jq .

# 检查 WASM 文件
ls -la .output/chrome-mv3/wasm/
```

### 加载扩展到浏览器

#### Chrome

1. 打开 `chrome://extensions/`
2. 开启右上角的 **开发者模式**
3. 点击 **加载已解压的扩展程序**
4. 选择对应目录（如 `.output/chrome-mv3/`）

#### Edge

1. 打开 `edge://extensions/`
2. 开启右上角的 **开发者模式**
3. 点击 **加载已解压的扩展程序**
4. 选择 `.output/chrome-mv3-edge/`

#### Firefox

1. 打开 `about:debugging#/runtime/this-firefox`
2. 点击 **临时加载扩展程序**
3. 选择 `.output/firefox-mv2/manifest.json`

#### Safari

1. 打开 Xcode
2. 选择 **File** → **Open**
3. 打开 `apps/browser-extension` 文件夹
4. 选择 Safari 扩展的 scheme
5. 点击运行
6. 在 Safari 中启用：**Safari 设置** → **扩展** → 开启 Tex2Doc

### 4. 自动化测试脚本

创建测试验证脚本 `scripts/test-extension.mjs`：

```javascript
import { chromium } from 'playwright';

async function testExtension() {
  const browser = await chromium.launch({ headless: false });
  const context = await browser.newContext();
  const page = await context.newPage();

  const results = {
    passed: [],
    failed: [],
  };

  try {
    // 1. 测试弹出窗口打开
    console.log('Testing: Popup opens...');
    await page.goto('https://example.com');
    // ... 测试代码 ...

    results.passed.push('Popup opens');

  } catch (error) {
    results.failed.push({ test: 'Popup opens', error: error.message });
  }

  await browser.close();

  console.log('\n=== Test Results ===');
  console.log(`Passed: ${results.passed.length}`);
  console.log(`Failed: ${results.failed.length}`);

  if (results.failed.length > 0) {
    console.log('\nFailed tests:');
    results.failed.forEach((f) => console.log(`- ${f.test}: ${f.error}`));
    process.exit(1);
  }
}

testExtension();
```

### 4. 版本号管理

扩展版本在以下位置定义：

- `apps/browser-extension/package.json` → `version` 字段
- `apps/browser-extension/wxt.config.ts` → manifest 配置
- 构建输出会自动使用此版本号

遵循语义化版本：`major.minor.patch`

- `major`: 主版本变更（不兼容的 API 修改）
- `minor`: 次版本变更（向后兼容的功能新增）
- `patch`: 修订版本（向后兼容的问题修复）

### C. 调试技巧

1. **查看 Service Worker 日志**
   - Chrome/Edge: 右键扩展图标 → 检查 → Application → Service Workers
   - 点击 "console" 链接查看日志

2. **查看存储数据**
   - 右键扩展图标 → 检查 → Application → IndexedDB
   - 查看 `tex2doc_extension` 数据库

3. **网络请求调试**
   - 打开扩展的 Network 面板
   - 查看 API 请求和响应

4. **模拟后台重启**
   - MV3 Service Worker 可能被终止
   - 在 Chrome DevTools 中手动终止以测试恢复逻辑
