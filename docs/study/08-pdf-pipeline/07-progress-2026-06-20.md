## 7.2 Semantic TeX Engine facade

新增 `doc-compiler-engine`，核心类型：

```rust
SemanticTexEngine
CompileOptions
CompileArtifact

CompileReport
EngineProfile
```

公开入口：

```rust
compile_source_to_docx(main_tex, source, options)
compile_dir_to_docx(project_root, main_tex, options)
compile_zip_to_docx(zip_bytes, main_tex_path, options)
compile_vfs_to_graph(main_tex, vfs, options)
compile_vfs_to_docx(main_tex, vfs, options)
```

编译阶段：

```text
SourceMount
IncludeGraph
TexParse
SemanticCollect
DocumentGraph
DocxRender
```

当前实现复用 `doc-latex-reader` 和 `doc-docx-writer`，但已把 `DocumentGraph` 和阶段报告作为稳定边界。后续 LuaHook collector、XDV layout collector、AI macro fallback、HTML/Markdown renderer 都应优先挂到该 facade。

## 7.3 paper3 to-docx 脚本

新增脚本：

```bash
bash scripts/build_paper3_compiler_engine_docx.sh
```

输出：

```text
examples/paper3/output/to-docx/v13-论文稿件-jos-<timestamp>-compiler-engine.docx
```

脚本内部调用：

```bash
cargo run -p doc-compiler-engine --example paper3_to_docx -- \
  --project-root examples/paper3/latex \
  --main-tex examples/paper3/latex/main-jos.tex \
  --profile jos-paper \
  --out examples/paper3/output/to-docx/<name>.docx
```

已验证输出特征：

```text
docx bytes: about 3.0 MB
blocks: 250
image-assets: 10
stages: SourceMount / IncludeGraph / TexParse / SemanticCollect / DocumentGraph / DocxRender all Completed
```

DOCX zip 中包含：

```text

word/styles.xml
word/settings.xml
word/_rels/document.xml.rels
word/media/image1.png ... image10.png
word/header*.xml / word/footer*.xml
```

## 7.4 V2 CLI

`doc-engine` CLI 位于 `crates/cli`：

```bash
cargo run -p doc-engine -- --help
```

子命令：

| 子命令 | 用途 |
|---|---|
| `convert` | zip → DOCX，沿用 `doc-core` |
| `tex-compile` | TeX → oracle PDF |
| `docx-to-pdf` | DOCX → PDF |
| `verify-pdf` | 三层质量对比并输出报告 |
| `build` | 一键串联 convert、tex-compile、docx-to-pdf、verify-pdf |
| `ast-dump` | 输出标准文档 AST |
| `render-dump` | 输出 DOCX 渲染树 |
| `docx-diff` | 对比两个 DOCX |

典型转换：

```bash
cargo run -p doc-engine -- convert \
  --zip examples/paper3/upload.zip \
  --main-tex main-jos.tex \
  --page-setup jos-paper3 \
  --out examples/paper3/output/to-docx/paper3-rust.docx
```

完整闭环：

```bash
cargo run -p doc-engine -- build \
  --zip examples/paper3/upload.zip \
  --main-tex main-jos.tex \
  --latex-main main-jos.tex \
  --page-setup jos-paper3 \
  --outdir examples/paper3/output/to-docx
```

## 7.5 pandoc 对照路径

新增：

```bash
scripts/build_paper3_pandoc_docx.sh
```

用途：

- 把 paper3 LaTeX 展开为 pandoc 友好的单一 `master.tex`。
- 递归展开 `\input{...}`。
- 改写 `\includegraphics{...}` 为绝对路径。
- 生成 `*-pandoc.docx` 作为对照基线。

`scripts/build_paper3_dual_docx.sh` 已接入该路径：若系统存在 `pandoc`，会在 sh/rust 双产物之外额外生成 pandoc 产物；若不存在则跳过，不阻塞主流程。

## 7.7 下一步建议

1. 把 `doc-core::convert_dir/convert_zip` 逐步迁移为调用 `doc-compiler-engine`，保留旧 API。
2. 为 `DocumentGraph` 增加显式 `ReferenceGraph`，结构化 `label/ref/cite`。
3. 把 JOS/中文学术/医学模板 profile 外置为规则表。
4. 为 `doc-engine build` 增加 paper3 专用脚本或 Make target，减少手动参数。
5. 将 LuaHook/XDV collector 作为独立 crate 骨架接入 `SemanticTexEngine`。
