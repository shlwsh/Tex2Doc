# 文档转换引擎实现算法总结

> 版本：v1.0 | 日期：2026-06-17

## 1. 项目整体架构

Doc-engine 是一个将 LaTeX 学术文档转换为 Word (`.docx`) 的 Rust 工作区项目，核心设计目标是对接软件学报（JOS）投稿模板，保留期刊版式、页眉页脚、引用格式、中英文混排等学术出版特性。

### 1.1 Crate 分工

| Crate | 职责 |
|-------|------|
| `core` | 统一对外门面：`convert_sync` / `convert_dir` / `convert_zip`，协调各模块 |
| `latex-reader` | LaTeX 源码解析：include 拓扑、宏展开、词法、语法、归一化、降级 |
| `semantic-ast` | 强类型语义模型：`Document` / `Block` / `TextRun` / `TextStyle` |
| `mathml` | LaTeX 数学公式转 OMML（Word 数学标记） |
| `docx-writer` | 语义 AST → OOXML 序列化 + ZIP 打包成 docx |
| `tex-facade` | 可选外部 TeX 编译器后端（Tectonic/XeLaTeX/LaTeXmk） |
| `docx-pdf` | 可选 docx → PDF 后端（LibreOffice） |
| `quality` | 可选 PDF 质量校验（视觉/结构/文本） |
| `bib` | BibTeX / bbl 解析 |
| `utils` | 公共工具：VFS、路径解析、图像资产 |
| `wasm` / `server` / `cli` / `native` | 多端入口 |

### 1.2 核心 Pipeline

```
IncludeGraph 拓扑
    ↓ join (Kahn 拓扑排序)
JoinedStream 单流文本
    ↓ expand_macros (VFS 感知宏展开)
Expanded Text
    ↓ lexer (Logos) → parser (Rowan GreenNode)
CST (Concrete Syntax Tree)
    ↓ lower_to_document
Semantic AST (Document)
    ↓ serialize_document + pack_with_page_setup
.docx (ZIP/OOXML)
```

## 2. Pass-1：Include 拓扑（`latex-reader/src/include.rs`）

### 2.1 算法目标

LaTeX 项目通常由 `main.tex` + 多个 `\input{chapter.tex}` / `\include{section.tex}` 组成，且图片路径由 `\graphicspath{{figs/}}` 声明。转换引擎首先需要把分散的 `.tex` 文件按依赖关系拼接成**单流文本**，同时收集所有图片资源。

### 2.2 核心算法

#### 2.2.1 DAG 构建（BFS + 哈希去重）

```
IncludeGraph::build(vfs, main)
    queue ← [main]
    stack ← [(main, {main})]
    while queue 非空:
        file ← pop_front
        scan_includes(file) → [(cmd, target)]
        for cmd, target in includes:
            if cmd == graphicspath:
                解析路径列表 → graphics_paths
            else:
                resolver.resolve(vfs, target) → hit
                if hit in stack 中任意集合:
                    Cycle(hit)
                if hit 未录入图:
                    add_node(hit)
                    stack.push((hit, {hit}))
                    queue.push_back(hit)
                edges[file] += hit
```

关键数据结构：
- `sources: Vec<PathBuf>` — SourceId → 路径
- `by_path: HashMap<PathBuf, SourceId>` — 路径 → SourceId
- `edges: HashMap<SourceId, Vec<SourceId>>` — 邻接表

#### 2.2.2 Kahn 拓扑排序

```
topo_order():
    indeg ← 所有节点入度初始为 0
    for tos in edges.values():
        for t in tos: indeg[t]++
    queue ← 入度为 0 的节点
    while queue 非空:
        n ← pop_front
        out.push(n)
        for t in edges[n]:
            indeg[t]--
            if indeg[t] == 0: queue.push_back(t)
    if out.len() != sources.len(): Cycle
    return out
```

#### 2.2.3 单流拼接

`IncludeGraph::join()` 按拓扑序遍历，将每个源文件内容用 `\n` 连接，同时生成 `source_map: Vec<SourceId>`（每个字符对应哪个源文件），供后续宏展开阶段做 VFS 感知。

### 2.3 复杂度

- 构建：O(V + E)，V 为文件数，E 为 include 边数
- 排序：O(V + E)
- 单次扫描完成，无回溯

## 3. 宏展开（`latex-reader/src/expand.rs`）

### 3.1 算法目标

LaTeX 模板大量使用 `\newcommand` 定义缩写（如 `\rjtitle`、`\AbstractContentZh`），需要在不执行 TeX 引擎的前提下，把宏调用替换为宏体。

### 3.2 核心策略

#### 3.2.1 两阶段展开（VFS 感知）

```
expand_macros_with_input(joined, vfs, macros):
    result ← ""
    i ← 0
    while i < text.len():
        if 命中 \input / \include:
            解析 {file}
            resolve_vfs_file(vfs, file) → content
            sub_macros ← 递归展开 content
            macros.extend(sub_macros)
            result += sub_text + "\n"
        else if 命中宏定义:
            parse_definition → 记入 macros，整行跳过
        else if 命中已知宏调用:
            result += macros[cmd]
        else:
            result += 当前字符
    return result
```

关键设计：
- **单遍扫描**：从前向后逐字节推进，遇到 `\` 启动命令识别
- **单词边界保护**：`\Xtra` 不会误展开为 `\X` + `tra`（检查下一个字节是否为 ASCII 字母/`@`）
- **VFS 递归**：`\input{file}` 会递归读取 VFS 中的子文件，收集其中的宏定义到全局 `MacroMap`
- **避免重复展开**：当 `joined` 已包含拼接后的内容时，只做纯宏展开（不再处理 `\input`），防止正文翻倍

#### 3.2.2 宏定义解析

```
parse_definition_end(text, pos, macros):
    跳过空白 → 找到 {name}
    跳过可选 [n]
    找到 {body}（支持跨行、depth=0 时跳过 % 注释）
    macros.define(name, body)
    return body 末尾 } 之后位置
```

### 3.3 局限

- 不支持 `\def` / `\let` / 条件宏
- 不支持带可选参数 `[default]` 的宏体参数替换（`#1` 占位符不展开）
- 不支持局部作用域（group scope）

## 4. 词法分析（`latex-reader/src/lexer.rs`）

### 4.1 工具

使用 **Logos** 生成词法分析器，定义 `TokKind` 枚举：

```rust
#[derive(Logos)]
pub enum TokKind {
    #[regex(r"\\[A-Za-z@]+")] Command,
    #[token("{")] LBrace,
    #[token("}")] RBrace,
    #[token("$")] Dollar,          // 行内数学
    #[token("$$")] DollarDollar,   // 块级数学
    #[regex(r"%[^\n]*")] Comment,
    #[regex(r"[ \t]+")] Whitespace,
    #[regex(r"\r?\n")] Newline,
    #[regex(r"\\\\")] LineBreak,
    #[token(r"\par")] Par,
    Error,  // 兜底：任意 UTF-8 字符
}
```

### 4.2 设计要点

- **极简文法**：只识别 Token 级别，不做上下文判断
- **容错性**：无法识别的字符统一归为 `Error`，后续解析器可处理
- **数学定界符保留**：`$` / `$$` 作为特殊 token，供后续提取公式使用

## 5. 语法分析（`latex-reader/src/parser.rs`）

### 5.1 工具

使用 **Rowan** 的 `GreenNodeBuilder` 构建不可变语法树（Green Tree），实现"无文法硬编码"的极简解析。

### 5.2 核心算法

```
parse_into(text, builder):
    brace_depth ← 0
    env_stack ← []
    for (tok, span) in TokKind::lexer(text).spanned():
        match tok:
            LBrace:
                start_node(Group)
                brace_depth++
            RBrace:
                if brace_depth > 0:
                    finish_node()
                    brace_depth--
                else:
                    start_node(Error)  // 多余 } 不 panic
                    token(RBrace)
                    finish_node()
            Command:
                if starts_with "\begin":
                    start_node(Env)
                    env_stack.push(Env)
                    token(Begin)
                elif starts_with "\end":
                    if env_stack.pop():
                        finish_node()
                    else:
                        start_node(Error)
                        token(End)
                        finish_node()
                else:
                    token(Command)
            Whitespace → token(Whitespace)
            Newline/LineBreak/Par → token(TokNewline)
            Comment → token(Comment)
            Dollar/DollarDollar → token(MathInline)
            Error → token(Error)
    // 收尾：自动闭合未闭合的 Group / Env
    while brace_depth > 0: finish_node(); brace_depth--
    while env_stack.pop(): finish_node()
```

### 5.3 设计要点

- **错误恢复**：未闭合的大括号/环境自动补一个虚拟闭合，绝不 panic，保证下游总能得到合法 CST
- **保留原始文本**：`Parse.source` 保存完整输入字符串，供 Lowering 阶段做原始文本切片
- **Source Map**：`JoinedStream.source_map` 记录每个字符所属的源文件，用于错误定位

## 6. 文本归一化（`latex-reader/src/normalize.rs`）

### 6.1 算法目标

LaTeX 源码包含大量排版命令（`\textbf`、`\cite`、`\ref`、`$math$`、`\footnote` 等），而 Word 文档需要纯文本 + 样式标记。归一化模块负责把 LaTeX 片段"翻译"为 docx 可接受的富文本结构。

### 6.2 核心数据结构

```rust
pub struct NormalizedText {
    pub runs: Vec<NormalizedRun>,
}
pub struct NormalizedRun {
    pub text: String,
    pub style: TextStyle,  // Plain/Bold/Italic/Code/MathInline/Superscript/Subscript
}
```

### 6.3 26 步归一化流水线（与 Python 版字节级等价）

```
latex_to_text(text, cite_map, label_map):
    1.  strip_comments      — 删除 % 注释（处理奇数 \ 转义）
    2.  CR → LF
    3.  \\ → 空格
    4.  \, → 空格
    5.  \{ / \} → 占位符 \u{FFF0}/\u{FFF1}
    6.  \cite{key} → [N,M-N]（compress_numbers 压缩连续编号）
    7.  \ref{label} → label_map 值
    8.  \label{...} → ""
    9.  $...$ / \(...\) → clean_math(content)（$$...$$ 保留）
    10. \footnote{...} → "（注：内容）"
    11. \textbf / \textit / \emph / \texttt / \mathrm
        → sentinel 包裹（\u{0001}B 等），后续 split_runs 切分
    12. \item[LABEL] → "LABEL "
    13. \item → ""
    14. `` / '' → 中文引号 “ / ”
    15. --- / -- → 长/短破折号
    16. \% / \& / \_ / \# / \$ → 字面量
    17. ~ → 空格
    18. 字体/字号宏删除（\xiaowuhao / \song / \kai 等）
    19. \fontsize{...}{...}\selectfont → ""
    20. \hspace / \vspace → " "
    21. 通用兜底：\\[A-Za-z]+\*?(?:\[[^\]]*\])? → 保留内容或删除
    22. 剥外层 {}（递归，最多 6 轮）
    23. 占位符还原
    24. 多空白 → 单空格
    25. trim
    26. split_runs_with_sup_sub — 切分为带样式的 run
```

### 6.4 关键技术：Sentinel 机制

为避免正则回溯，`\textbf{...}` 等样式命令被替换为 `\u{0001}B...text...\u{0001}B` 包裹，在最终 `split_runs_with_sup_sub` 阶段识别 sentinel 并切分为对应 `TextStyle` 的 run。这种方式比递归正则更可控，且支持嵌套样式。

### 6.5 关键技术：上标/下标切分

`split_runs_with_sup_sub` 在归一化后字符串上扫描：
- `[N]` / `[1-3]` / `[N,M]` → `Superscript`（启发式：首字符为数字且其余为数字/逗号/连字符）
- `^X` / `^{XYZ}` → `Superscript`
- `_X` / `_{XYZ}` → `Subscript`（但 `_code` 中下划线后接 ASCII 字母视为代码，保留）

## 7. 降级：CST → Semantic AST（`latex-reader/src/lower.rs`）

### 7.1 算法目标

将 CST + 归一化结果转换为强类型的 `Document { metadata, blocks }`，消融所有 LaTeX 语法特性为标准块枚举。

### 7.2 顶层扫描策略

```
lower_with_macros_and_numbering(parse, joined, macros, numbering):
    text ← joined.text 或 parse.source
    text ← expand_macros(text, macros)   // 宏展开
    text ← strip_preamble(text)           // 跳过 \begin{document} 之前

    pos ← 0
    while pos < text.len():
        skip_whitespace_and_comment
        if scan_environment(pos) 命中:
            flush_paragraph
            if env 是多块容器 (flushleft/center/verbatim 等):
                递归降级 body → sub blocks
            else:
                lower_environment(name, body) → Block
                doc.push(block)
            pos = end
        elif try_top_level_command(pos) 命中:
            flush_paragraph
            doc.push(Heading / Paragraph)
            pos += consumed
        elif \bibliography:
            flush_paragraph
            doc.push(Block::Paragraph { text: "References" })
            pos += 行长度
        elif \rjkeywords:
            flush_paragraph
            doc.push("关键词" label + content)
        elif detect_section_label:
            flush_paragraph
            doc.push(label paragraph)
        elif 元数据命令 (rjtitle/fancyhead 等):
            pos += consumed  // 直接吞掉，不产出块
        else:
            line ← 取到 \n
            stripped ← strip_inline(line)  // 行内清洗
            buffer += stripped + "\n"
            pos = 行尾

    flush_paragraph  // 收尾

    // 提取 front matter
    fm ← extract_front_matter(parse.source, text, macros)
    doc.metadata ← fm
```

### 7.3 环境降级

`lower_environment` 处理特定环境：

| 环境 | 输出 Block |
|------|-----------|
| `itemize` / `enumerate` / `description` | `Block::List { is_ordered, items }` |
| `tabular` / `array` | `Block::Table { rows, caption }` |
| `figure` / `table` | `Block::Figure` / `Block::Table` |
| `equation` / `equation*` | `Block::Equation { latex, is_block }` |
| `algorithm` | `Block::Algorithm { lines, io, caption }` |
| `rjabstract` / `rjkeywords` | 带 Bold label 的 Paragraph |
| 其他 | `Block::RawFallback`（不 panic，保留原文） |

### 7.4 Algorithm2e 解析（`latex-reader/src/algorithm.rs`）

```
parse_algorithm_rows(body):
    iter ← LogicLineIter(body)   // 按 \; / \n / } 切逻辑行
    indent_stack ← [0]
    for line in iter:
        strip_algorithm_line_meta   // 剥 \KwIn / \KwOut / \caption
        keyword ← detect_keyword    // ForEach/If/For/While/Return/Else
        match keyword:
            ForEach/If/For/While:
                cond, body ← split_two_braces
                out.push(AlgLine { indent, guides, code: "kw (cond)" })
                sub ← parse_algorithm_rows(body)  // 递归
                for s in sub: s.indent += 1
                out.extend(sub)
                out.push(AlgLine { indent: close, code: "end" })
                indent_stack.pop()
            Return:
                out.push(AlgLine { code: "return val" })
            _:
                out.push(AlgLine { code: normalize_alg_code(raw) })
```

算法块设计：
- **缩进栈**：嵌套 `ForEach` / `If` 时 `indent_stack` 递增/递减
- **竖线指引**：`guides` / `end_guides` 记录悬挂缩进线位置（目前 Rust 版简化保留字段，未实际渲染竖线）
- **注释提取**：`\tcp*{comment}` 被抽离为 `AlgLine.comment`

## 8. 序列化：Semantic AST → OOXML（`docx-writer/src/`）

### 8.1 整体流程

```
pack_with_page_setup(doc, template_bytes, image_assets, page_setup):
    // 1. 解析模板
    template ← parse_template(template_bytes)
    styles ← merge_styles(template.base_styles, doc)

    // 2. 序列化 document.xml
    embedded_images ← []
    document_xml ← serialize_document(doc, image_assets, page_setup, &mut embedded_images)

    // 3. 构建 ZIP 包
    zip ← new()
    zip.add("[Content_Types].xml")
    zip.add("word/document.xml", document_xml)
    zip.add("word/styles.xml", write_styles(styles))
    if page_setup 有 header/footer:
        zip.add("word/header1.xml", ...)
        zip.add("word/footer1.xml", ...)
    for img in embedded_images:
        zip.add("word/media/imageN.ext", img.bytes)
    return zip.into_inner()
```

### 8.2 关键序列化逻辑

#### 8.2.1 样式映射

| Semantic AST Block | JOS 样式 |
|-------------------|----------|
| `Heading { level: 1 }` | `JOSHeading1` |
| `Heading { level: 2 }` | `JOSHeading2` |
| `Heading { level: 3 }` | `JOSHeading3` |
| `Paragraph` (正文) | `JOSBodyText` |
| `Paragraph` (参考文献) | `JOSReference` (悬挂缩进) |
| `List` (有序) | `JOSListNumber` |
| `List` (无序) | `JOSListBullet` |
| `Table` cell | `JOSTableText` |
| `Algorithm` | `JOSCode` (Courier New) |
| `Equation` | `JOSCode` + OMML |

#### 8.2.2 图片嵌入

```
for Block::Figure { path, caption }:
    key ← path (POSIX)
    if image_assets 命中 key:
        bytes ← image_assets[key]
        ext ← png/jpg/jpeg
        fig_id ← 自增
        embedded_images.push(EmbeddedImage { fig_id, ext, bytes })
        写入 <w:drawing> + <a:blip r:embed="rIdN"/>
    else:
        写入占位文本 "[图: path]"
```

#### 8.2.3 Header/Footer 字段

支持 `{{PAGE}}` / `{{NUMPAGES}}` 占位符，渲染为 Word 域代码：

```xml
<w:fldChar w:fldCharType="begin"/>
<w:instrText xml:space="preserve"> PAGE </w:instrText>
<w:fldChar w:fldCharType="separate"/>
<w:t>1</w:t>
<w:fldChar w:fldCharType="end"/>
```

### 8.3 数学公式

公式块使用 `doc-mathml`  crate 将 LaTeX 转为 OMML（Office Math Markup Language），嵌入 `<w:altChunk>` 或直接写入 `<m:oMathPara>`。

## 9. 设计思路与 Trade-off

### 9.1 架构原则

1. **Reader / Writer 完全解耦**：`semantic-ast` 作为中间 IR，Reader 只产出 `Document`，Writer 只消费 `Document`，两端可独立演进
2. **VFS 抽象**：所有文件访问走 `VirtualFs`，支持本地目录、内存 ZIP、WASM 沙箱等多源
3. **容错优先**：未闭合括号、未识别命令、缺失图片均走 `RawFallback` 或占位文本，绝不 panic
4. **字节级兼容**：归一化流水线与 Python 参考实现字节级等价，保证与既有 PDF 输出一致

### 9.2 已知局限与演进方向

| 局限 | 当前状态 | 未来改进 |
|------|---------|----------|
| 宏展开 | 不支持 `\def` / 参数替换 | 引入完整 TeX 宏引擎或嵌入 TeX 编译器 |
| 语法解析 | 极简文法，无完整 LaTeX 语法 | 可引入 Lalrpop 构建更精确 parser |
| 算法块竖线 | `guides` 字段保留但未渲染 | 需在 serializer 中生成 `<w:tbl>` + 竖线 |
| 表格 | 基础行列支持，无合并单元格 | 扩展 `TableCell` 模型 |
| 数学公式 | OMML 基础支持 | 更复杂公式需测试 Office 兼容性 |
| 样式 | 21 个 JOS 硬编码样式 | 支持模板动态样式继承 |
